MarkDown basic syntaxis

MarkDown is a lenguage that allow give format at plain text using in platform as GitHub, Obsidian notes, etc. The great advantage of MarkDown is that it tries to achieve maximum readability and ease of publication both in its input and output form and allows you to create websites, documents, notes, books, presentations, emails and documentation.

In general, MarkDown it’s marvelous to begginer in CS that have interest in obtain text ordened and with big quality by way of simple and left the WYSIWG (What you see is what you get) paradigm that implements apps as Microsoft Word and Google docs.

MarkDown has grown since its launch in 2004 to acquire different functionalities although in the following article I will focus on the Basic syntax for a MarkDown programmer.

Headlings

As always it is good to be hierarchical MarkDown allows you to divide the sections through up to 6 levels of headling. The syntax is simple you use the number of numerals (a.k.a hashtag) equal to the level you want followed by a space and you can write your headling !

# H1 ## H2 ### H3 #### h4 ##### h5 ###### h6

Fun fact: Levels 1 and 2 have an alternate syntax

Headling 1 ========= Headling 2 --------

Emphasis

We all like to model and style certain words to give them more power and distinguish them from the rest. To achieve this huge distinction MarkDown allows two: bold and italic

In a very brief way so as not to tire you, dear reader. If you want bold you can do it in 2 ways:
